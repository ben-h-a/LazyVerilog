# LazyVerilog v1.2.4 Release Notes

## Bug Fixes
- Fixed race condition where the parser thread could overwrite an in-flight text change. Parsing is now worker-based instead of debounce-based.
- Fixed AutoFF formatting the entire file instead of only the changed region (issue #4).
- Fixed stale formatting in several edge cases (issues #5, #6, #7).

## Go-to-Definition Improvements
- Functions and tasks inside packages are now indexed. Go-to-def on them resolves correctly for closed project files (.f filelist).
- `SyntaxIndex` is now shared across snapshots instead of deep-copied on every `didChange`, reducing per-edit memory overhead.

## Config
- Removed `lint.diagnostic_debounce_ms` option (no longer needed with worker-based parsing).
