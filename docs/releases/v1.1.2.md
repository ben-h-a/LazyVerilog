# LazyVerilog v1.1.2 Release Notes

Formatter safety and save-time AutoArg improvements.

## Changes

This release adds a stricter formatter validation mode and makes save-time AutoArg use formatter safe-mode automatically. The new `format.safe_mode2` option re-lexes formatted output and compares token streams, which helps catch accidental tokenization changes that whitespace-only checks would miss. AutoArg save-on-write behavior also inherits formatter safe-mode so generated module headers fail safely if formatting would otherwise alter non-whitespace content.

## Fixes

- Added `format.safe_mode2` to compare token streams before and after formatting.
- `autoarg_on_save` now forces formatter `safe_mode` during save-time generated edits.
- Documented the new formatter and AutoArg safety behavior.
