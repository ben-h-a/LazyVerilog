# LazyVerilog v1.0.3 Release Notes

## New Features

### AutoArg on Save

Non-ANSI module port lists can now be regenerated automatically on every file save.

Enable in `lazyverilog.toml`:

```toml
[autoarg]
autoarg_on_save = true
```

When enabled, LazyVerilog regenerates the `(port, list, ...);` header for every module in the saved file whose ports are declared in the module body. The port list is formatted according to `[format.module]` settings and then passed through the formatter if `enable_format_on_save` is also enabled.

This removes the need to manually invoke the AutoArg code action after adding or removing ports.
