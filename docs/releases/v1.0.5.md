# LazyVerilog v1.0.5 Release Notes

Completion-focused release.

## Changes

LazyVerilog now provides broader SystemVerilog autocompletion, including context-aware keywords, visible identifiers, member access, package scope, named ports/parameters, macros, snippets, include files, `new`, and event-control contexts. Completion is now mostly driven by slang SyntaxTree context, so suggestions are filtered by scope, imports, package membership, connected ports, and the current syntactic context. Package/member completion also works better across files, including after jumping to a package or class definition and returning.

## Fixes

- `vcode.f` now supports simulator-style `+incdir+...` include directories.
- slang built-in macros such as `SV_COV_ERROR` are hidden from completion, hover, and go-to-definition.
- Class type names now resolve to their class declarations in go-to-definition.
- Standalone package, interface, class, and typedef files are indexed consistently with compilation-unit members.
- Fixed `:Interface` argument handling.
